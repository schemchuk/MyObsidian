# AppArmor — короткий конспект

## Загальна інформація

**AppArmor** — це Linux-модуль обовʼязкового контролю доступу (MAC, Mandatory Access Control), який використовується в **Debian-based системах**:

- Debian
    
- Ubuntu
    
- Kali Linux
    
- Parrot OS
    

Керування AppArmor виконується **від імені користувача root**.

---

## Стани AppArmor

AppArmor має **три основні стани профілів**:

- **enforcing**
    
- **complain**
    
- **disabled**
    

---

### enforcing

- правила безпеки **перевіряються**
    
- заборонені дії **блокуються**
    
- події **логуються**
    

Використовується в продуктивних середовищах.

---

### complain

- правила безпеки **перевіряються**
    
- заборонені дії **НЕ блокуються**
    
- відбувається **логування порушень**
    

Використовується для тестування та навчання.

> ⚠️ `complain` **не дорівнює** `permissive` (це термін із SELinux).

---

### disabled

- політики безпеки **не застосовуються**
    
- контроль доступу **відсутній**
    

---

## Перевірка стану AppArmor

```
apparmor_status
```

Команда показує:

- загальний стан AppArmor
    
- список профілів
    
- які профілі в `enforcing` або `complain`
    

---

## Розташування профілів

Профілі AppArmor зберігаються в каталозі:

```
/etc/apparmor.d/
```

Приклад профілю для бінарника `man`:

```
/etc/apparmor.d/usr.bin.man
```

> AppArmor працює **по шляхах до виконуваних файлів**.

---

## Керування режимами профілів

Для роботи з профілями потрібен пакет:

```
apt install apparmor-utils
```

### Увімкнути enforcing

```
aa-enforce /usr/bin/man
```

### Увімкнути complain

```
aa-complain /usr/bin/man
```

### Вимкнути профіль

```
aa-disable /usr/bin/man
```

> Команди застосовуються до **бінарника**, а не до файлу профілю напряму.

---

## Важливі зауваження

- AppArmor — шлях-орієнтований (path-based) MAC
    
- `complain` ≠ `permissive`
    
- Повне вимкнення AppArmor у системі потребує параметрів ядра
    

---

## Підсумок

- AppArmor — MAC для Debian-based систем
    
- 3 режими: `enforcing`, `complain`, `disabled`
    
- Профілі зберігаються в `/etc/apparmor.d/`
    
- Керування через `apparmor-utils`
