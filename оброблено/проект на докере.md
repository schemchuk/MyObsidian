# сборка и запуск Docker контейнеров на моем проекте : 

- **Выполняю команду для сборки Docker-образа**:
docker build -t url-shortener-app .
- **Запускаю контейнер** 
 docker run -p 8080:8080 --name url-shortener-app url-shortener-app
- **Проверяю, что контейнер запущен и работает**:
docker ps
- **Останавливаю все контейнеры:**
docker stop $(docker ps -q)
можно остановить по имени  или по номеру 
docker stop <CONTAINER_ID или CONTAINER_NAME>

Чтобы почистить Docker-среду после выполнения всех операций, следуйте этим шагам:

### 1. **Остановить и удалить все контейнеры**

Сначала остановите все запущенные контейнеры:

`sudo docker stop $(sudo docker ps -q)`
sudo docker stop $(sudo docker ps -q)

Затем **удалите** все контейнеры, включая остановленные:

`sudo docker rm $(sudo docker ps -a -q)`
sudo docker rm $(sudo docker ps -a -q)

### 2. **Удалить все образы**

Удалите все Docker-образы с вашей системы:

`sudo docker rmi $(sudo docker images -q)`

sudo docker rmi $(sudo docker images -q)

### 3. **Удалить неиспользуемые данные**

Для удаления всех неиспользуемых данных, включая остановленные контейнеры, неиспользуемые образы, сети и 

`sudo docker system prune -a --volumes`
sudo docker system prune -a --volumes

Команда `--volumes` также удаляет все неиспользуемые тома, освобождая больше места на диске.
