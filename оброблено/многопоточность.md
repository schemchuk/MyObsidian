многопоточность нужна для распаралеливания потоков, не нужно ждать выполнения предыдущей инструкции как в однопоточном режиме

  ##**недостатки:**
  -  состояие **гонки**  *concurent_Colision*  когда несколько потоков работают с одними и теми же данными
 - **dedlog** логическая коллизия когда потоки взаимно блокируются
 - правильная организация логических потоков

планировщик процессов распределяет выполнение процессов в процессоре. работает за счет увеличения частоты процессора. Это длилось до определенного времени.
сейчас используются многоядерные процессоры 
[[поток]] наименьшая единица обработки информации исполнение которой может быть назначено ядром операционной системы .это легкий процесс. **потоки** запускаются внутри **процесса** 

**[[поток]]** в джаве имеет четыре состояния 
- NEW - новый поток создан, обэявлен но **не** запущен
- RUNNING - состояниу после запуска потока выполняет свою работу
- WAITING  может иметь три разновидности:  
         - **blockt** поток вынужден ждать каких-то данных
         - ***waiting*** ожидает окончания работы другого потока
         - **teimwaiting** ожидает только определенное время, потом завершает работу
-  DEAD состояние после завершения или возникновения какой-то ошибки

процесс некая программа которая выполняется в данное время и все что для этого необходимо. [[процессы]] между собой **независим**
