##      инструкция от чата
Вот исправленная и более точная инструкция по поднятию и связыванию приложения и базы данных Redis с Docker.

## Поднятие и связывание приложения и базы данных

### 1. Вход в терминал нужного приложения

Откройте терминал и перейдите в директорию вашего приложения.

### 2. Поднятие Redis

Redis — это база данных в памяти. Мы запускаем ее с пробросом порта.

`docker run -d -p 6379:6379 redis`

Эта команда запускает контейнер с образом Redis в фоновом режиме и пробрасывает порт 6379 из контейнера на хост.

### 3. Проверка запущенного контейнера Redis

Чтобы посмотреть список запущенных контейнеров и убедиться, что Redis работает:
`docker container ls`

### 4. Получение IP-адреса контейнера Redis

Теперь нам нужно узнать IP-адрес контейнера Redis, чтобы приложение могло к нему подключиться.

1. Найдите контейнер Redis в списке и скопируйте его идентификатор (ID).
    
2. Выполните команду для получения информации о контейнере:
    
    bash
    
    Copy code
    
    `docker inspect <container_id>`
    
    Замените `<container_id>` на ID контейнера Redis. В выводе найдите строку `"IPAddress"`, которая содержит IP-адрес контейнера, например:
    
    [[json]]
    
    Copy code
    
    `"IPAddress": "172.17.0.5",`
    

### 5. Настройка приложения

Прежде чем запускать приложение, убедитесь, что оно настроено на использование Redis. В файле `app.py` установите следующие переменные окружения:
`REDIS_HOST = os.environ["REDIS_HOST"] REDIS_PORT = os.environ["REDIS_PORT"]`

### 6. Сборка образа приложения

Теперь нужно сбилдить Docker-образ для вашего приложени
`docker build -t lab2:0.1 .`

Эта команда создает Docker-образ с именем `lab2` и версией `0.1` из Dockerfile в текущей директории.

### 7. Запуск контейнера с приложением

Чтобы запустить контейнер с вашим приложением и связать его с Redis, выполните следующую команду:
`docker run -d -p 3000:8888 -e REDIS_HOST=172.17.0.5 -e REDIS_PORT=6379 lab2:0.1`

Здесь:

- `-d` — запускает контейнер в фоновом режиме.
- `-p 3000:8888` — пробрасывает порт 8888 из контейнера на порт 3000 на хосте.
- `-e REDIS_HOST=172.17.0.5` — передает IP-адрес контейнера Redis в переменную окружения `REDIS_HOST`.
- `-e REDIS_PORT=6379` — передает порт Redis в переменную окружения `REDIS_PORT`.
- `lab2:0.1` — это имя образа, который мы собираем и запускаем.

### 8. Проверка работы приложения

Откройте браузер и перейдите по адресу `http://localhost:3000`, чтобы проверить работу вашего приложения. Оно должно быть связано с Redis и использовать его как базу данных.

### Примечания

- Убедитесь, что все зависимости и конфигурации приложения правильно настроены для работы с Redis.
- Если вы используете Docker Compose, вам не нужно вручную связывать контейнеры по IP-адресам, так как Docker Compose автоматически организует сеть между контейнерами.

Эта инструкция поможет вам правильно настроить и связать ваше приложение с базой данных Redis в Docker.
