## Аналіз мереж за допомогою IFCONFIG
**ifconfig** є одним з найбільш базових інструментів для вивчення і взаємодії з активними мережевими інтерфейсами.
## Перевірка пристроїв бездротової мережі за допомогою IWCONFIG
**iwconfig**  для збору важливої інформації для бездротового злому, такої як IP-адреса адаптера, його MAC-адреса, режим роботи тощо. Інформація, яку ви можете отримати з цієї команди, особливо важлива, коли ви використовуєте бездротові інструменти злому, такі як **aircrackng**.
### **Перегляд IP-адреси**
**ip addr show** — загальний перегляд IP-адрес:
 **ip addr show dev eth0** - щоб побачити адресу конкретного інтерфейсу:
  **ip addr show dev wlan0 -** щоб побачити адресу конкретного інтерфейсу:
  **ifconfig** — старий спосіб перегляду:
  **hostname -I** — для швидкого перегляду IP-адреси:
  **curl ifconfig.me** — для перегляду публічної IP-адреси:
### Зміна IP-адреси
### **Тимчасова зміна IP-адреси**
Призначення нової IP-адреси для інтерфейсу (наприклад, `eth0`):
ip addr add 192.168.181.115/24 dev eth0
Видалення IP-адреси з інтерфейсу:
ip addr del 192.168.181.115/24 dev eth0
Використання `ifconfig` (старіший спосіб):
ifconfig eth0 192.168.181.115 netmask 255.255.255.0 up
Видалення IP-адреси з інтерфейсу:
ifconfig eth0 del 192.168.181.115

## Перевірка DNS за допомогою dig
## Основні можливості команди `dig`

### Отримання інформації про сервери імен (NS)

Щоб отримати список серверів імен (nameservers) для домену, використовуйте параметр ns
dig hackersarise.com ns
dig hackersarise.com ns
### Отримання інформації про поштові сервери (MX)

Щоб дізнатися про сервери обміну поштою (mail exchange servers), використовуйте параметр mx
dig hackersarise.com mx
dig hackersarise.com mx
### Перевірка IP-адреси домену (A-запис)

Щоб отримати IP-адресу домену (A-запис), просто введіть:
dig hackersarise.com
dig hackersarise.com

### Перевірка всіх доступних записів (ANY)
dig hackersarise.com any
dig hackersarise.com any
### Отримання SOA-запису (Start of Authority)

SOA-запис вказує основну інформацію про домен.
dig hackersarise.com soa
### Вказання конкретного DNS-сервера для запиту

Можна перевірити DNS-запис через конкретний сервер імен:
dig @8.8.8.8 hackersarise.com
dig @8.8.8.8 hackersarise.com
**`@8.8.8.8`** — використовує публічний сервер Google DNS.
## Примітки

1. **BIND (Berkeley Internet Name Domain):** Найпоширеніший DNS-сервер у Linux. Його часто називають "BIND", хоча це лише реалізація DNS-сервера.
2. `dig` надає повний контроль над запитами DNS і є основним інструментом для тестування та діагностики DNS.
