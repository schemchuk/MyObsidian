JPA (Java Persistence API) — это спецификация для управления сохранением данных между Java-объектами (POJO) и реляционными базами данных. JPA позволяет разработчикам использовать объектно-реляционное отображение (ORM) для связывания объектов Java с таблицами базы данных без необходимости писать SQL-запросы непосредственно.


связь между базой данных и сущностями определяет **DAO** (data acscess objekt)- специальный интерфейс в котором определены методы взаимодействия. Частным случаем является класс **Repositopy** в JPA



Основные компоненты JPA:

1. **Entity**: Класс, который представляет собой таблицу в базе данных. Каждый экземпляр класса соответствует строке в таблице.
2. **Entity Manager**: Интерфейс для выполнения операций с сущностями, таких как сохранение, обновление, удаление и поиск.
3. **Persistence Unit**: Группа классов-сущностей, которые управляются одним Entity Manager.
4. **Query Language (JPQL)**: Язык запросов, подобный SQL, но работающий с сущностями, а не с таблицами и строками.

Основные аннотации JPA:

- `@Entity`: Определяет класс как сущность.
- `@Table`: Указывает, к какой таблице в базе данных привязана сущность.
- `@Id`: Обозначает первичный ключ сущности.
- `@GeneratedValue`: Указывает стратегию генерации значений для первичного ключа.
- `@Column`: Настраивает свойства столбца базы данных, соответствующего полю сущности.
- `@OneToMany`, `@ManyToOne`, `@OneToOne`, `@ManyToMany`: Описывают отношения между сущностями.
