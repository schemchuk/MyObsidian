Чтобы переключиться обратно на основную ветку (например, `main` или `master`) в Git, сохраняя при этом свои изменения и не влияя на основную ветку, выполните следующие шаги:

1. **Проверьте текущий статус вашего репозитория**: Убедитесь, что у вас нет несохраненных изменений, которые могут потеряться при переключении веток.
    
    git status
    
2. **Сохраните ваши изменения с помощью stash**: Если у вас есть незакоммиченные изменения, которые вы не хотите потерять, можно временно сохранить их с помощью `git stash`.
    
    git stash
    
    Это сохранит ваши изменения и очистит рабочую директорию. Вы сможете восстановить их позже.
    
3. **Переключитесь на основную ветку**:
    
    git checkout main  # или git checkout master
    , если ваша основная ветка называется master`
    
4. **Создайте новую ветку для сохранения ваших изменений**: Чтобы сохранить ваши изменения отдельно, создайте новую ветку от основной ветки.

    git checkout -b my-feature-branch
    
5. **Восстановите ваши изменения из stash**:
    git stash pop
    
6. **Закоммитьте ваши изменения в новую ветку**:
    
    git add . git commit -m "Сохранение моих изменений в новую ветку"
    

Таким образом, вы сохраните ваши изменения в новой ветке и не повлияете на основную ветку.

### Альтернативный подход (без использования stash)

Если вы не хотите использовать stash, вы можете закоммитить ваши изменения в текущую ветку перед переключением:

1. **Закоммитьте ваши изменения**:

    git add . git commit -m "Сохранение моих изменений"
    
2. **Переключитесь на основную ветку**:
    
    git checkout main  
     или
    git checkout master
    
3. **Создайте новую ветку от основной ветки**:
4. 
    git checkout -b my-feature-branch
    
Теперь ваши изменения будут сохранены в новой ветке, и основная ветка останется неизменной.
