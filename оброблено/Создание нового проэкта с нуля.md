Заходим в идею и создаем новый проэкт ( в даном случае на мавен архетипе)
производим сборку проэкта :
открываю справ вкладку [[Maven]] захожу в 
TestMavenArchetype -> Lifecycle
и здесь начинаю по порядку запускать вкладки. Clean  не со всеми запускается , можно иногда пропустить.
остальные вкладки при запуске запускают предыдущие
запуск производится в окошке возле зеленого треугольника

если тесты не проходят их можно отключить аннотацией  **Dizable** 

после операции **package**    мы получаем джарник - .jar файл в папке **target** 

после запуска **verify** and **install** джарник будет установлен в  новую папку на **локальном** репозитории
приблизительно так выглядит  Installing /home/volodymyr/myMavenProgects/TestMavenArchetype/pom.xml to /home/volodymyr/.m2/repository/com/TestMavenArchetype/1.0-SNAPSHOT/TestMavenArchetype-1.0-SNAPSHOT.pom

практически выполняется операция копирования созданных файлов на локальном компе в локальный  **maven_репозиторий** 
#### чаще всего запускают не по одной операции , а сразу 
### mvn clean instal
или два раза кликнуть по install

для запуска джарника из командной строки  терминала заходим в папку **target**  например так:      cd /home/volodymyr/myMavenProgects/TestMavenArchetype/target
и запускаем:         java -jar TestMavenArchetype-1.0-SNAPSHOT.jar
если нет **манифест** файла то устанавливаем его

### Отредактировать файл манифест
захожу в папку **таргет**:  cd /home/volodymyr/myMavenProgects/TestMavenArchetype/target
заходим в манифест:      nano MANIFEST.MF 

!!!если ничего не видно то   nano META-INF/MANIFEST.MF





### Пример полного процесса:

1. Создайте файл `MANIFEST.MF`:
    
    sh
    
    Copy code
    
    `echo "Manifest-Version: 1.0" > MANIFEST.MF echo "Main-Class: com.example.Main" >> MANIFEST.MF`
    
2. Создайте новый JAR файл с обновленным манифестом: 
 
    
    sh
    
    Copy code
    
    `jar cfm TestMavenArchetype-1.0-SNAPSHOT.jar MANIFEST.MF -C . .`
    
3. Запустите новый JAR файл:
    
    sh
    
    Copy code
    
    `java -jar TestMavenArchetype-1.0-SNAPSHOT.jar`
    

Если вы используете Maven для сборки вашего проекта, вы можете указать главный класс в вашем `pom.xml`, чтобы Maven автоматически добавлял его в манифест при сборке JAR файла.

Добавьте следующие строки в ваш `pom.xml`:

xml

Copy code

`<build>     <plugins>         <plugin>             <groupId>org.apache.maven.plugins</groupId>             <artifactId>maven-jar-plugin</artifactId>             <version>3.2.0</version>             <configuration>                 <archive>                     <manifest>                         <addClasspath>true</addClasspath>                         <classpathPrefix>lib/</classpathPrefix>                         <mainClass>com.example.Main</mainClass>                     </manifest>                 </archive>             </configuration>         </plugin>     </plugins> </build>`

Замените `com.example.Main` на ваш главный класс. Затем выполните команду сборки:

sh

Copy code

`mvn clean package`

После этого запустите ваш JAR файл:

sh

Copy code

`java -jar target/TestMavenArchetype-1.0-SNAPSHOT.jar`
