### [[генерация ключей SSH]]

**демони** перевіряють і зєднують приватний ключ з публічним

переходим в корневой раздел 
**cd** 
потом команда
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
Замість `your_email@example.com` вкажіть вашу електронну адресу, з якою ви хочете пов'язати цей ключ
потом **три** раза нажимаем **enter**  
после чего ключи будут сгенерированы

Теперь ваши SSH-ключи сгенерированы. Публичный ключ (`id_rsa.pub`) и приватный ключ (`id_rsa`) будут находиться в папке `~/.ssh/`.

1. После завершения процесса у вас будут два файла:
    
    - **Приватный ключ:** `~/.ssh/id_rsa`
    - **Публичный ключ:** `~/.ssh/id_rsa.pub`
    ## достать ключ
    cat ~/.ssh/id_rsa.pub

1. **Добавьте публичный ключ на сервер:** Чтобы использовать созданные ключи для аутентификации, вам нужно добавить содержимое файла `id_rsa.pub` в вкладке на гите :
заходим на гит, выбираем во вкладке Settings поле с ключиком называется ssh and GPS keys и вставляем свой ключ. В поле title даем название ключа обычно по имени девайса.  Если есть другие лишние то удаляем их

Потом в домашнем терминале можем сделать команды
git remote -v
git push -u origin main
и выполнить подсказки а можно

## диспетчер задач [[процессы]] 

**top**  показує всі процеси в режимі реального часу
**ps -ef**  процеси які були вже запущені на момент запуску **ps** 
**ps -ef | wc -l** порахувати кількість запущених процесів на момент запуску **ps**
**ps -ef | grep ssh** виборка всіх процесів які були вже запущені по **ssh** -тобто всіх усерів підключених по технології **ssh** 

на початку виводу стоїть процес демон який перевіряє сумісність публічного і приватного ключів усера. позначений буквою  :  -D

## Другие команды управления процессами
**pstree** Выводит список процессов в виде древовидной структуры, отражающей от-
ношения «родитель–потомок» между процессами
**vmstat**  Выводит мгновенный снимок с информацией об использовании системных
ресурсов, включая память, файл подкачки и объем дискового ввода/вывода.
Чтобы увидеть, как изменяется эта информация с течением времени, пере-
дайте команде интервал задержки (в секундах) между обновлениями (напри-
мер, vmstat 5). Завершить работу команды можно нажатием CTRL+C
**xload**  Программа с графическим интерфейсом, показывающая изменение нагрузки
на систему с течением времени
**tload**  Работает подобно программе xload, но рисует график в терминале. Заверша-
ется работа команды нажатием CTRL+C
