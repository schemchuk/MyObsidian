### Техническое Задание для Проекта URL Shortener

#### 1. Введение

Проект представляет собой сервис для сокращения URL-адресов, который включает в себя управление пользователями и ролями, а также функции для создания и управления короткими URL-адресами. Сервис реализован с использованием Java и [[Spring]] Boot, а также поддерживает работу с базой данных через Hibernate/[[JPA]].

#### 2. Архитектура и Компоненты

- **Frontend**: Не включен в проект (если потребуется, его можно добавить отдельно).
- **Backend**: Реализован с использованием Spring Boot.
- **База данных**: Используется для хранения данных о пользователях, ролях и коротких URL.
- **Мапперы**: Конвертируют объекты между уровнями приложения и DTO.
- **Контроллеры**: Обрабатывают [[HTTP]]-запросы и взаимодействуют с сервисами.
- **Сервисы**: Содержат бизнес-логику приложения.

#### 3. Функциональные Требования

1. **Управление пользователями**:
    
    - Регистрация пользователей с уникальным именем пользователя и электронной почтой.
    - Изменение данных пользователя (имя, электронная почта, пароль).
    - Удаление пользователя.
    - Получение информации о пользователе по ID.
2. **Управление ролями**:
    
    - Получение информации о роли по имени.
    - Обновление роли по ID.
3. **Сокращение URL**:
    
    - Создание короткого URL для полного URL.
    - Получение полного URL по короткому ключу.
    - Увеличение счётчика кликов по короткому URL.
    - Удаление короткого URL.
4. **Аутентификация и авторизация**:
    
    - Пользователи могут иметь различные роли, такие как "TRIAL", которые могут быть использованы для управления доступом.

#### 4. Классы и [[интерфейсы]]

- **DTO (Data Transfer Object)**:
    
    - `RoleRequest`: [[запрос]] для создания или обновления роли.
    - `RoleResponse`: [[ответ]] с информацией о роли.
    - `ShortUrlRequest`: Запрос для создания короткого URL.
    - `ShortUrlResponse`: Ответ с информацией о коротком URL.
- **Entity**:
    
    - `User`: Сущность пользователя.
    - `Role`: Сущность роли.
    - `ShortUrl`: Сущность короткого URL.
- **Repository**:
    
    - `UserRepository`: Репозиторий для управления пользователями.
    - `RoleRepository`: Репозиторий для управления ролями.
    - `ShortUrlRepository`: Репозиторий для управления короткими URL.
- **Service**:
    
    - `UserService`: Интерфейс и реализация для управления пользователями.
    - `RoleService`: Интерфейс и реализация для управления ролями.
    - `ShortUrlService`: Интерфейс и реализация для управления короткими URL.
- **Controller**:
    
    - `RoleController`: Контроллер для управления запросами, связанными с ролями.
    - `UserController`: Контроллер для управления запросами, связанными с пользователями.
    - `ShortUrlController`: Контроллер для управления запросами, связанными с короткими URL.

#### 6. Документация API

- **GET /roles/{roleName}**: Получение информации о роли по имени.
- **PUT /roles/{id}**: Обновление информации о роли по ID.
- **POST /users**: Регистрация нового пользователя.
- **PUT /users/{id}**: Обновление информации о пользователе по ID.
- **DELETE /users/{id}**: Удаление пользователя по ID.
- **POST /short-urls**: Создание короткого URL.
- **GET /short-urls/{shortKey}**: Получение полного URL по короткому ключу.
- **PUT /short-urls/{shortKey}/increment-clicks**: Увеличение счётчика кликов по короткому URL.
- **DELETE /short-urls/{id}**: Удаление короткого URL по ID.
