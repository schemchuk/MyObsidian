додаємо устройства які нам потрібно, звязуєм кабелями і переходимо до розподілення адресного простору. вказуємо класові елементи які будемо використовувати в експериментах.
далі розподіляю ІР адреси за інтерфейсами.
заходжу в інтерфейс командної строки роутера 1 , повводивши паролі, далі заходжу в режим глобальної конфігурації
en
conf t
interface fastEthernet 0/1
no shutdown
ip address 192.168.10.1 255.255.255.0
do wr
do sh run подивитися налаштування
заходжу в інтерфейс командної строки роутера 0
Пристрій новий в мережі але не буду налаштовувати авторизацію.
заходжу в режим глобальної конфігурації
en
conf t
hostname R0 -   задаю імя хоста
no ip domain-lookup
no shutdown
ip address 192.168.10.2 255.255.255.0
do wr
exit
interface fastEthernet 0/0
no shutdown
ip address 192.168.50.1 255.255.255.0
do wr
do sh run подивитися налаштування
налаштовуємо кінцевий пристрій - сервер. клацаю по ньому і через декстоп заходжу в ІР configuration, де налаштовую інтерфейс вказуючи ІР адресу сервера і роутера0

далі переходжу в любий комп і пінгую сервер 
ping 192.168.50.10
але отримую повідомлення що віддалений вузол недоступний.
 переходжу на маршрутизатор роутер1
 заходжу в режим глобальної конфігураціі
 en
 conf t
 і прописую маршрут
 ip route 192.168.50.0 255.255.255.0 192.168.10.2
 do wr
 do sh run подивитися налаштування
 exit
 show ip route подивитися маршрути - **таблию маршрутизації**
 заходжу на комп пінгую ще раз і бачу що знаходиться в режимі очікування.
 це получилось тому що маршрутизатор роутер0 не знає куди давати відповідь
 налаштовую маршрут з маршрутизатора роутер0.
