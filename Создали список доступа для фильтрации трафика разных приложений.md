робллю налаштування кінцевих точок далі переходжу на маршрутизатори
## R0
##### стандартні
- hostname R0
- no ip domain lookup
- line console 0
- logging synchronous
- exit
#### далі по інтерфейсах

int fa 0/0
no sh
ip address 192.168.101.1 255.255.255.0
int fa 0/1
no sh
ip address 100.10.101.1 255.255.255.252
## R1
##### стандартні
- hostname R0
- no ip domain lookup
- line console 0
- logging synchronous
- exit
#### далі по інтерфейсах

int fa 0/1
no sh
ip address 100.10.101.2 255.255.255.252
int fa 0/0
no sh
ip address 10.1.101.1 255.255.255.0
do wr
## переходжу до налаштувань протоколу **OSPF** на маршрутизаторах

**R0**
router ospf 10
passive-interface fa 0/0
Подивитись маршрути:
do sh ip ro
внутрішні інтерфейси маршрутизаторів переводяться в стан **passiv**
network 100.10.101.0 0.0.0.3 area 0
network 192.168.101.0 0.0.0.255 area 0
do wr

**R1**
router ospf 1
passive-interface fa 0/0
do sh ip ro
network 10.1.101.0 0.0.0.255 area 0
network 100.10.101.0 0.0.0.3 area 0
do wr

налаштовую DNS-server
Можна протестувати.
можна перейти до написання списку досеупу
**R1**

ip access-list extended Servers
permit ospf any any

permit icmp any host 10.1.101.2

permit tcp any host 10.1.101.3 eq www

permit tcp any host 10.1.101.4 eq ftp
дозволяю взаємодію по групі портів
permit tcp any host 10.1.101.4 range 1024 1124

дозволяю роботу по протоколах ftp  та www
permit tcp any host 10.1.101.5 range ftp www
дозволяю взаємодію по групі портів 
permit tcp any host 10.1.101.5 range 1024 1124

do wr

решта зєднаннь забороняється
deny ip any any
do wr
подивитися список доступу
в реж гл. конф
do sh run
#### привязка списку доступу до інтерфейсу fa 0/1

int fa 0/1
ip access-group Servers in         (Servers це назва листа , in - фільтрація вхідного трафіку  )
do wr
