# Ефективне керування дозволами доступу в Linux

## Базові права доступу
- **r** — читання (read)
- **w** — запис (write)
- **x** — виконання (execute)

## Категорії користувачів
- **u** — власник (user)
- **g** — група (group)
- **o** — інші (others)
- **a** — всі (all)

## Управління правами
```bash
chmod 755 file.txt        # Власник rwx, група та інші r-x
chmod u+x file.txt        # Додати право виконання власнику
chmod g-w file.txt        # Забрати право запису в групи
```

## Спеціальні біти
- **SUID** (Set User ID) — програма виконується з правами власника.
- **SGID** (Set Group ID) — програма/каталог виконується/створює файли з групою власника.
- **Sticky bit** — у каталозі файл може видаляти лише власник.

## Розширені ACL
```bash
setfacl -m u:alex:rw file.txt   # Додати користувачу alex права читання та запису
getfacl file.txt                # Переглянути ACL
```

## Рекомендації
- Використовувати принцип найменших привілеїв
- Регулярно перевіряти `ls -l` і `getfacl`
- Мінімізувати використання SUID/SGID для безпеки
