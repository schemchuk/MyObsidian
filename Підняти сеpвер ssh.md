3   Теги: , [[Підняти сеpвер ssh]]
працюю від рута 
ставлю програмне забезпечення на машині сервері
apt install openssh-server

перевіряю
systemctl status ssh

далі на машині клієнті
apt openssh-client
приконектуюся до сервера
ip asroot
ssh lessons@192.168.1.63
yes
pass      ввести пароль користувача

ip a    перевіряю чи дійсно я на тій машиnі що хотів
поміняти настройку порта чи інші настройки доступу можна в конфігураційному файлі:
/etc/ssh/sshd_config 

копіювання з клієнта на сервер і навпаки можливе якщо задати можливість доступу клієнта (нехороша практика) на сервер від імені рута.
використовується команда :
scp    а не cp



при роботі з  ssh доцільно використовувати **ключі**
генерація ssh ключів:
ssh-keygen
відправляю публічний ключ на сервер
заходжу в папку .ssh  і з неї виконую команду: 
ssh-copy-id lessons@192.168.1.207
ss
конектитися :
ssh lessons@192.168.1.207
