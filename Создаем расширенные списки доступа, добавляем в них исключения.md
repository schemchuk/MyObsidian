## Заблокувати ICMP запити до серверів , при цьому доступ по протоколах  [[HTTP]] i FTP вільний 
**R1**
створюю розширений список доступу:

ip access-list extended EXServers01

блокувати застосунок, пакети якого не повинні проникати в цю мережу:
deny icmp any any    - (заблоковати всі ICMP пакети від усіх усім)
permit tcp any any  -(явно дозволити доступ інших застосунків (HTTP i FTP, які являються TCP застосунками ) протокол TCP від усіх усім  )
do wr
### виконую привязку access-list  -а до інтерфейсу , який обслуговує веб сервери (int fa 0/0)
**R1**
int fa 0/0
ip access-group EXServers01 out
do wr
## Нова задача: добавити в VLAN1 новий компютер, щоб він міг використовувати запити ICMP

добавляю компютер і налаштовую його.
додаю налаштування на комутаторі:
**SW0**
int fa 0/11
switchport mode access
switchport access vlan 101
do wr
**R1**
складу список доступу в якому буде **виключення** з правил
ip access-list extended EXServers02 (складаю новий аксес ліст)
permit icmp host 192.168.101.101 any (записую виняток)
deny icmp any any (заборона всім решті користуватися ICMP)
permit tcp any any (дозвіл на використання протоколів TCP від усіх усім)
do wr
ex
відвязати попередній аксес ліст (access-list extended EXServers01) від обслуговуючого інтерфейсу fa 0/0 і привязати до нього новий аксес ліст:
int fa 0/0
no ip access-group EXServer01 out
ip access-group EXServer02 out
do wr
