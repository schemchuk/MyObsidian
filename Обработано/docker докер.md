У описанных ниже команд есть общий синтаксис, который выглядит так: **docker container** _**название команды**_. Названия, собственно, ниже:

- **create** — создает контейнер из выбранного образа.
- **start** — активирует уже существующий контейнер. обязательно должна крутиться какая то задача иначе контейнер остановится и в него нельзя будет зайти. при поытке сделать **restart** команда отработает но будут утрачены свежие данные
- **restart** = stop + start
- **run** — создает новый контейнер и сразу включает его. **Не** путать с запуском контейнера **docker run --** 
потому что она используется только для запуска образа
- **ls** — отображает все существующие контейнеры.
- **inspect** — подробнее рассказывает о выбранном контейнере.
- **logs** — выводит в консоль логи (то есть журнал событий).
- **stop** — пытается остановить выбранный контейнер, отправив ему сигнал **SIGTERM**, требующий завершить всю активность и сохранить пользовательские данные. Если [[ответ]] занимает слишком много времени, то следом посылает сигнал **SIGKILL**, чтобы «убить» процесс без сохранения данных.
- **docker rm**  и  **stop** всегда идут рядом удаляет выбранный контейнер (он должен быть выключен, чтобы команда сработала).
- **kill** — выполняет ту же задачу, что и предыдущая команда, но пропускает шаг с отправкой **SIGTERM**. Сразу выключает контейнер, игнорируя сохранение пользовательских данных.
- **rm** — удаляет выбранный контейнер (он должен быть выключен, чтобы команда сработала). если ругается то применяем ключ **-f**   или **--fors** для принудительного удаления
- **docker commit** делает коммит (сохранение изменений)
- **docker pause** возможность поставить на паузу и потом вернуть обратно **docker unpause** 
-  **docker unpause** потом вернуть обратно
- **docker pull**  скачать образ

Для удаления всех Docker-образов и контейнеров, выполните следующие команды:

### 1. **Остановить все работающие контейнеры**

`docker stop $(docker ps -aq)`
docker stop $(docker ps -aq)

Эта команда остановит все запущенные контейнеры.

### 2. **Удалить все контейнеры**


`docker rm $(docker ps -aq)`
docker rm $(docker ps -aq)

Команда удалит все контейнеры, как остановленные, так и запущенные.

### 3. **Удалить все Docker-образы**

`docker rmi $(docker images -q)`
docker rmi $(docker images -q)

Эта команда удалит все образы Docker.

### 4. **Удалить неиспользуемые объекты (необязательно)**

Если вы хотите очистить также и неиспользуемые данные, такие как сети, тома и dangling-образы, используйте команду:

`docker system prune -a`
docker system prune -a

Эта команда удаляет все остановленные контейнеры, неиспользуемые образы, тома и сети.
