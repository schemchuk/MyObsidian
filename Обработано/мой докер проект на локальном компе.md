# подготовка к запуску докера
1. создаю докер файл Dockerfile в котором приложение собирается прямо в контейнере. Это дольше по времени но зато позволит запускать контейнеры на любой машине , и не нужно будет собирать проект на локальной машине. 
2. мне нужно запустить два контейнера (для базы данных и приложения), Для етого надо использовать **docker-compose.yml** 
3. создается файл .**dockerignore** 

# сборка и запуск Docker контейнеров на моем проекте : 
### перед запуском на локальном компютере отключить mysql  :

sudo systemctl stop mysql    **важно!!!!!!!!!!!!**

*или* 
sudo service mysql stop

*проверить что никто не занимает порт*
sudo lsof -i :3306

- **Выполняю команду для сборки Docker-образа**:
docker-compose build
- **Запускаю контейнеры с помощью Docker Compose:**
docker-compose up
*Если возникли проблемы, попробуйте удалить существующие контейнеры и создать их заново:*
**удалить все контейнеры** 
docker-compose down -v
**создать заново**
docker-compose up --build

   **Удалить все образы**
sudo docker rmi $(sudo docker images -q)

- **Проверяю, что контейнеры запущены и работают**:
docker ps
- **Останавливаю контейнеры:**
docker-compose down

Если вы обнаружите, что **MySQL сервер уже** работает на вашем компьютере, вы можете остановить его, если он не нужен:

sudo service mysql stop

или

sudo systemctl stop mysql

# запустить обратно mysql: 

sudo service mysql start
или
sudo systemctl start mysql

### 3. **Удалить неиспользуемые данные**

sudo docker system prune -a --volumes
