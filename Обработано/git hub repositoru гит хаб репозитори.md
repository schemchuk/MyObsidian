#### создание репозитория 
[[git hub repositoru гит хаб репозитори]] 

для проверки связанности гита с ненужными директориями на раб столе и вообще выще уровнем той папки где еме быть положено:
git rev-parse --show-toplevel
сделать папку на рабочем столе

команда чтоб узнать текущее размещение проэкта
pvd

при соединении локального и глобального репозиториев выполняем команды которые написаны в глобальном.

когда добавляем файлы для коммита то после  **git add .** выполняем команду которая поможет отследить добавляемые файлы и подсказки покажут как удалить ненужные  **git status**
но лучше выкидывать файлы из отслеживания командой **git reset**  (название файла)

если надо делать переименование ветки с master na main: 
git branch -M main

привязка делается при помощи команды которая есть там же 
git remote add origin git@github.com:schemchuk/telegramBotProject.git
git branch -M main
git push -u origin main

проверка привязки 
git remote -v
должно показать возможность делать команды 
fetch and push

## подходы для работы:
## git hub flow

имеем две ветки для разработки :
-  основная (main)
- ветка для тестирования( как хочеш назови)
## git  flow  
имеем три ветки для разработки :
-  основная (main)
- ветка в которую переносятся **все** изменения на всякий случай 
- ветка для тестирования( как хочеш назови)
